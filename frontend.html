语法对比 — Dart vs JavaScript / TypeScript

Dart 的类型系统和 TypeScript 非常接近，都是强类型。主要区别在于 Dart 多了 const（编译时常量）和 dynamic（≈ any）。

变量与类型

JS JavaScript / TypeScript
TS
const name: string = "hello";  // 不可重新赋值
let count: number = 0;        // 可重新赋值
let old = "legacy";            // 类型推断

const API = "https://api.com"; // 运行时常量

let age: number = 25;
let data: any = "anything";   // 任意类型

// 基本类型
number, string, boolean
Array<string>, Record<string, any>
⇄ 差异说明

JS const ≈ Dart final（运行时常量）

JS 没有编译时常量 → Dart const（编译时常量）

JS let = Dart var（类型推断后可变）

TS any ≈ Dart dynamic

JS number → Dart 分 int 和 double

Dart 字符串用单引号 'hello'

Dart 有字符串插值 '$name'（≈ 模板字符串）

D Dart
DART
final String name = 'hello';  // 运行时常量
int count = 0;               // 可重新赋值
var old = 'legacy';          // 类型推断，锁定

const String API =           // 编译时常量
  'https://api.com';

int age = 25;
dynamic data = 'anything';   // 任意类型

// 基本类型
int, double, bool, String
List<String>, Map<String,dynamic>

Dart 函数需声明返回类型（TS 也推荐）

TS

03

空安全 — Dart 的 Null Safety

TypeScript 用 ? 和 ?? 处理空值，Dart 的空安全和 TS 非常像，但更严格 — 编译器强制检查。

TS TypeScript
TS
// optional 类型
let name: string | undefined;
let title: string = "";

// nullish coalescing
const val = name ?? "default";

// optional chaining
const len = name?.length;

// non-null assertion
const n: string = name!;

// nullish assignment
name ??= "unknown";

// 接口可选属性
interface User {
  name: string;
  age?: number;
}
⇄ 差异说明

TS string | undefined → Dart String?

TS ?? = Dart ??（完全一样）

TS ?. = Dart ?.（完全一样）

TS ! = Dart !（非空断言）

TS ??= = Dart ??=（完全一样）

Dart 多了 late 延迟初始化

Dart 默认不可 null，编译器强制检查

D Dart
DART
// 类型后加 ? 表示可空
String? name;        // 可以为 null
String title = '';   // 不可为 null

// 空值合并
String val = name ?? 'default';

// 可选链
int? len = name?.length;

// 非空断言
String n = name!;

// 空值赋值
name ??= 'unknown';

// 延迟初始化
late String token;
// 用之前必须赋值，否则运行时报错

⏰ late 关键字 — TS 没有的概念

late 告诉编译器："这个变量我保证在使用前会赋值"。常用于 Controller 中需要在 onInit() 里初始化的字段。如果忘了赋值就使用，运行时会报 LateInitializationError。

DART
class UserController extends GetxController {
  late String token;  // 声明时不赋值

  @override
  void onInit() {
    super.onInit();
    token = getStoredToken();  // 初始化时赋值
  }
}
04

类 & 构造函数

TS 的 class 语法和 Dart 非常接近。但 Dart 多了命名构造函数、初始化列表、factory 等语法糖，写起来更简洁。

TS TypeScript
TS
class User {
  readonly name: string;
  readonly age: number;

  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }

  // 静态工厂方法
  static fromJson(json: any): User {
    return new User(json.name, json.age);
  }

  // getter
  get displayName(): string {
    return `${this.name} (${this.age})`;
  }
}
⇄ 差异说明

TS constructor → Dart this.name 语法糖

TS 只有一个 constructor → Dart 有命名构造函数

TS static fromJson → Dart User.fromJson()

TS get = Dart get（一样）

Dart 有初始化列表 :

Dart factory = 工厂模式内置

Dart 没有 public/private 关键字，用 _ 前缀表示私有

D Dart
DART
class User {
  final String name;
  final int age;

  // 语法糖：this.name 自动赋值
  User(this.name, this.age);

  // 命名构造 + 初始化列表
  User.fromJson(Map json)
    : name = json['name'],
      age = json['age'];

  // 命名参数构造
  User.create({
    required this.name,
    this.age = 0,
  });

  // getter
  String get displayName =>
    '$name ($age)';
}

🏭 factory 构造 = 单例模式内置

JS/TS 写单例需要 private 构造 + static instance。Dart 用 factory 一行搞定。

TSTypeScript 单例
class HttpManager {
  private static instance: HttpManager;

  private constructor() { }

  static getInstance(): HttpManager {
    if (!HttpManager.instance) {
      HttpManager.instance =
        new HttpManager();
    }
    return HttpManager.instance;
  }
}
DARTDart 单例
class HttpManager {
  static final HttpManager _instance =
    HttpManager._internal();

  factory HttpManager() => _instance;

  HttpManager._internal() {
    // 初始化 Dio...
  }
}

参数系统 — 命名参数 vs 位置参数

JS 用对象解构模拟命名参数，Dart 原生支持命名参数。这就是为什么 Flutter Widget 写起来像 JSX 的 props 但不用写接口定义。

JS JavaScript / TypeScript
TS
// 对象解构模拟命名参数
function createUser({
  name,
  age = 0,
  email,
}: {
  name: string;
  age?: number;
  email?: string;
}) { }

createUser({
  name: "Tom",
  age: 25,
  email: "tom@test.com",
});

// React props 也是对象解构
function Button({ label, onClick }: Props) {
  return <button onClick={onClick}>{label}</button>;
}
⇄ 差异说明

JS 解构 {} → Dart {} 命名参数

TS 需要定义接口 → Dart 直接在参数列表声明

required 标记必传

Dart [] = 可选位置参数

React props ≈ Flutter Widget 命名参数

这就是 Container(padding: ..., color: ...) 的由来

D Dart
DART
// 命名参数 — {} 包裹
void createUser({
  required String name,
  int age = 0,
  String? email,
}) { }

createUser(
  name: 'Tom',
  age: 25,
  email: 'tom@test.com',
);

// 可选位置参数 — [] 包裹
void log(String msg, [int level = 0]) {}
log('hello');
log('hello', 2);
为什么 Flutter Widget 全用 key: value 形式？
因为 Widget 构造函数用的就是 Dart 的命名参数 {}。就像 React 的 <Button label="OK" onClick={fn} />，Flutter 写成 ElevatedButton(onPressed: fn, child: Text('OK'))。本质一样，只是语法不同 — 一个是 JSX，一个是 Dart 构造函数。

级联操作符 & 扩展运算符

JS 有 spread 运算符 ...，Dart 也有。但 Dart 还多了一个级联操作符 ..，可以对同一个对象连续调用方法。

JS JavaScript / TypeScript
TS
// spread 运算符
const a = [1, 2];
const b = [...a, 3, 4];

const obj1 = { name: "Tom" };
const obj2 = { ...obj1, age: 25 };

// 链式调用（需要方法返回 this）
builder
  .setName("Tom")
  .setAge(25)
  .build();

// 没有级联操作符
const list: number[] = [];
list.push(1);
list.push(2);
list.push(3);
⇄ 差异说明

JS ... spread = Dart ...（数组/列表展开）

JS 对象 spread → Dart Map 也支持 ...

JS 没有级联 → Dart .. 级联操作符

.. 不需要方法返回 this

级联常用于初始化对象后连续设置属性

Dart 还有 ?. 空安全级联 ..

D Dart
DART
// spread 运算符（和 JS 一样）
var a = [1, 2];
var b = [...a, 3, 4];

var map1 = {'name': 'Tom'};
var map2 = {...map1, 'age': 25};

// 级联操作符 .. （JS 没有）
var list = <int>[]
  ..add(1)
  ..add(2)
  ..add(3);

// 常用于 Paint 等对象初始化
var paint = Paint()
  ..color = Colors.red
  ..strokeWidth = 2.0
  ..style = PaintingStyle.stroke;

异步进阶 — Future & Stream

JS 的 Promise 和 Dart 的 Future 几乎一一对应。Dart 还有 Stream，类似 JS 的 EventSource / RxJS Observable。

JS JavaScript / TypeScript
TS
// Promise + async/await
async function fetchUser(): Promise<User> {
  const res = await axios.get(url);
  return res.data;
}

// Promise.all 并发
const [a, b] = await Promise.all([
  fetchA(), fetchB()
]);

// .then 链式
fetchUser().then(user =>
  console.log(user.name));

// EventSource (SSE)
const es = new EventSource('/stream');
es.onmessage = (e) => {
  console.log(e.data);
};
⇄ 差异说明

Promise → Future

async/await 写法几乎一样

Promise.all → Future.wait

.then() → .then()（一样）

EventSource → Stream

onmessage → .listen()

Dart 有 async* + yield 生成 Stream

D Dart
DART
// Future + async/await
Future<User> fetchUser() async {
  final res = await dio.get(url);
  return User.fromJson(res.data);
}

// Future.wait 并发
final results = await Future.wait([
  fetchA(), fetchB()
]);

// .then 链式
fetchUser().then((user) =>
  print(user.name));

// Stream ≈ EventSource
Stream<String> sseStream() async* {
  yield 'chunk1';
  yield 'chunk2';
}
stream.listen((data) => print(data));

📦 Future = 单次异步结果

  • 等价于 JS 的 Promise
  • 一次请求，一次返回
  • 用 async/await 或 .then()
  • 场景：API 请求、文件读取、延时操作

🌊 Stream = 持续数据流

  • 等价于 JS 的 EventSource / RxJS Observable
  • 持续推送多个值
  • 用 .listen() 或 await for
  • 场景：SSE 对话、WebSocket、实时数据

状态管理对比

前端的状态管理你最熟了。Vue 的 ref/reactive、React 的 useState、Flutter 的 GetX — 核心思路都是：声明响应式变量 → 变量改变 → UI 自动更新。

V Vue Composition API
VUE
// composable
const count = ref(0);
const user = reactive({
  name: 'Tom',
  age: 25,
});

// 修改 → 自动更新 UI
count.value++;
user.name = 'Jerry';

// computed
const doubled = computed(
  () => count.value * 2
);

// watch
watch(count, (val) => {
  console.log('changed:', val);
});
R React Hooks
JSX
// useState
const [count, setCount] =
  useState(0);
const [user, setUser] =
  useState({ name: 'Tom' });

// 修改 → 自动更新 UI
setCount(count + 1);
setUser({ ...user, name: 'Jerry' });

// useMemo ≈ computed
const doubled = useMemo(
  () => count * 2, [count]
);

// useEffect ≈ watch
useEffect(() => {
  console.log('changed:', count);
}, [count]);
D Flutter GetX
DART
// .obs = 响应式变量
final count = 0.obs;
final user = {
  'name': 'Tom'
}.obs;

// 修改 → 自动更新 UI
count.value++;
user['name'] = 'Jerry';

// UI 中用 Obx 自动监听
Obx(() => Text(
  '${count.value}'));

// ever ≈ watch
ever(count, (val) {
  print('changed: $val');
});

// workers: debounce, interval...
debounce(count, (val) {
  print('debounced: $val');
});

🔄 StatefulWidget — Flutter 原生状态

  • 类似 React Class Component 的 setState
  • 需要手动调用 setState(() {}) 触发刷新
  • 适合简单的局部状态
  • 代码量较多，需要写两个类

⚡ GetX — 更像前端的方式

  • .obs ≈ Vue 的 ref()
  • Obx() ≈ Vue 的模板自动追踪
  • 不需要 setState，改值就刷新
  • 还自带路由、依赖注入、国际化

网络请求对比 — Axios vs Dio

前端用 Axios，Flutter 用 Dio。两者 API 设计非常像：都有拦截器、都支持请求/响应拦截、都能配置 baseURL。

JS Axios
TS
// 创建实例
const http = axios.create({
  baseURL: 'https://api.com',
  timeout: 10000,
});

// 请求拦截器
http.interceptors.request.use(
  (config) => {
    config.headers['Authorization']
      = `Bearer ${token}`;
    return config;
  }
);

// 响应拦截器
http.interceptors.response.use(
  (res) => res.data,
  (err) => {
    if (err.response?.status === 401) {
      router.push('/login');
    }
    return Promise.reject(err);
  }
);

// 使用
const res = await http.get('/users');
const res2 = await http.post('/users', data);
⇄ 差异说明

axios.create → Dio(BaseOptions(...))

interceptors.request → InterceptorsWrapper.onRequest

interceptors.response → InterceptorsWrapper.onResponse

err.response.status → DioException.response?.statusCode

API 设计几乎一一对应

Dio 还支持文件上传进度回调

Dio 支持 SSE 流式响应

D Dio
DART
// 创建实例
final dio = Dio(BaseOptions(
  baseUrl: 'https://api.com',
  connectTimeout:
    Duration(seconds: 10),
));

// 拦截器
dio.interceptors.add(
  InterceptorsWrapper(
    onRequest: (options, handler) {
      options.headers['Authorization']
        = 'Bearer $token';
      handler.next(options);
    },
    onError: (err, handler) {
      if (err.response?.statusCode
          == 401) {
        Get.offAllNamed('/login');
      }
      handler.next(err);
    },
  ),
);

// 使用
final res = await dio.get('/users');
final res2 = await dio.post('/users',
  data: data);

Widget 体系

前端同学可以这样理解：Flutter 的 Widget 就是 React/Vue 的组件。每个 UI 元素都是一个类的实例，通过构造函数的命名参数配置属性（≈ props）。

🧱 基础 Widget

  • Text — 文本（≈ <span>）
  • Image — 图片（≈ <img>）
  • Icon — 图标（≈ <i>）
  • ElevatedButton — 按钮（≈ <button>）
  • TextField — 输入框（≈ <input>）
  • Container — 容器（≈ <div>）
  • Scaffold — 页面骨架（≈ Layout 组件）

📐 布局 Widget

  • Row — 横向排列（≈ flex-direction: row）
  • Column — 纵向排列（≈ flex-direction: column）
  • Stack — 层叠（≈ position: relative/absolute）
  • ListView — 滚动列表（≈ v-for / .map()）
  • GridView — 网格（≈ display: grid）
  • Expanded — 弹性填充（≈ flex: 1）
  • Wrap — 自动换行（≈ flex-wrap: wrap）

TSPinia Store
// stores/user.ts
export const useUserStore = defineStore(
  'user', () => {
    const loading = ref(false);
    const userList = ref<User[]>([]);
    const error = ref('');

    async function fetchUsers() {
      loading.value = true;
      try {
        const res = await http.get('/users');
        userList.value = res.data;
      } catch (e) {
        error.value = e.message;
      } finally {
        loading.value = false;
      }

  }
);
DARTGetxController
class UserController extends GetxController {
  final isLoading = false.obs;
  final userList = <User>[].obs;
  final errorMsg = ''.obs;

  // ≈ onMounted
  @override
  void onInit() {
    super.onInit();
    fetchUsers();
  }

  Future<void> fetchUsers() async {
    isLoading.value = true;
    try {
      final res = await dio.get('/users');
      userList.value = (res.data as List)
        .map((e) => User.fromJson(e))
        .toList();
    } on DioException catch (e) {
      errorMsg.value = e.message ?? '失败';
    } finally {
      isLoading.value = false;
    }
  }
}
